### `station_daily_statistics` ‚Äî NULL line_id upsert fix

The `ON CONFLICT (date, station_id, line_id)` clause failed silently when `line_id` was NULL because PostgreSQL treats `NULL ‚âÝ NULL` in unique constraints.

Fix: dropped the named constraint, replaced with a functional unique index:
```sql
CREATE UNIQUE INDEX uq_station_daily_coalesce
ON station_daily_statistics (date, station_id, COALESCE(line_id, -1))
```

Same fix applied to `hourly_station_statistics`:
```sql
CREATE UNIQUE INDEX uq_hourly_station
ON hourly_station_statistics (hour_start, station_id, COALESCE(line_id, -1))
```

Updated `ON CONFLICT` clauses in `load.py` to use the expression form:
```sql
ON CONFLICT (date, station_id, COALESCE(line_id, -1)) DO UPDATE SET ...
```

## July 8‚Äì9 ‚Äî Recommendation Engine + DB Persistence

### `recommendation/dispatcher.py`

Rule-based engine that translates predictions into concrete operational actions.

**5 rules:**

| Priority | Trigger | Action |
|----------|---------|--------|
| üî¥ CRITICAL | occupancy ‚â• 95% | Deploy additional bus immediately |
| üüÝ HIGH + recurring | same slot HIGH 3+ days | Adjust permanent schedule |
| üüÝ HIGH | occupancy ‚â• 80% | Pre-position standby bus |
| üü° MEDIUM | occupancy ‚â• 60% | Alert driver |
| üü¢ LOW | occupancy < 20% | Consider reducing frequency |
| ‚ÑπÔ∏è INFO | tickets < passengers by >15% | Fare inspection alert |

**Commands:**
```bash
# Single station
python -m recommendation.dispatcher --station 3 --line 2 --hour 7 --hours 8

# Full network scan (HIGH and above)
python -m recommendation.dispatcher --network --hour 7 --hours 4 --min HIGH

# All including LOW
python -m recommendation.dispatcher --network --min LOW
```

### Recommendations persistence

Created `recommendations` table:

```sql
CREATE TABLE recommendations (
    recommendation_id   SERIAL PRIMARY KEY,
    created_at          TIMESTAMPTZ DEFAULT NOW(),
    prediction_date     DATE NOT NULL,
    line_id             INTEGER REFERENCES lines(line_id),
    station_id          INTEGER REFERENCES stations(station_id),
    hour                SMALLINT NOT NULL,
    predicted_occupancy NUMERIC(5,2),
    predicted_passengers INTEGER,
    severity            VARCHAR(20) NOT NULL,
    action              VARCHAR(100) NOT NULL,
    recommendation      TEXT NOT NULL,
    status              VARCHAR(20) NOT NULL DEFAULT 'pending'
)
```

Status values: `pending` / `acknowledged` / `resolved`

Dispatcher now auto-saves every recommendation to DB after each run. The full operational loop is:

```
CV Pipeline ‚Üí passenger_events
      ‚Üì
ETL ‚Üí analytics tables (hourly, daily, etc.)
      ‚Üì
Prediction ‚Üí expected_occupancy, expected_passengers
      ‚Üì
Dispatcher ‚Üí recommendations table (status: pending)
      ‚Üì
Power BI ‚Üí Operations & Recommendations page
```
